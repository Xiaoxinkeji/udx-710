## 1. 🏆 全栈成就系统
成就系统不再仅仅是前端的装饰，它通过 C 语言后端定时审计您的设备负载、信号质量与运行习惯，动态发放勋章。

- **如何查看**: 点击侧边栏的 **“🏆 成就”** 入口。
- **成就分类**:
    - **稳如泰山系列**: 奖励长达数天的无重启运行。
    - **流量大亨系列**: 奖励通过本设备吞吐的大规模数据流。
    - **信号猎人系列**: 奖励在极佳信噪比环境下的“黄金采样”。
- **技术细节**: 进度存储在本地 SQLite 数据库中，后端 `achievement.c` 每 60 秒进行一次全量判定。

## 2. 📡 蜂窝拓扑雷达
通过接入 `ofono` 的邻区监控接口，我们实现了一个动态的 SVG 径向布局雷达图，让周围基站的信号博弈“肉眼可见”。

- **如何查看**: 点击菜单中的 **“拓扑”**。
- **图表解读**:
    - **中心节点**: 您当前服务的主基站。
    - **四周节点**: 周围正在竞争或协同的邻区基站。
    - **距离/亮度**: 与信号强度 (RSRP/RSRQ) 正相关，越明亮的节点表示干扰或切换潜质越高。

## 3. 🤖 自动化流引擎
这是一个轻量级的 **IF-THEN** 规则引擎，允许设备在无人值守的情况下实现自愈与策略跳转。

- **如何配置**: 进入 **“自动化”** 面板。
- **触发器类型**:
    - `Temperature`: 基于 SoC 实时温度。
    - `Memory`: 基于系统可用内存在占比。
    - `Uptime`: 基于单次运行时间。
- **原生动作**:
    - `Reboot`: 触发硬重启。
    - `Drop Caches`: **[高级]** 原生释放系统缓存，不影响业务。
    - `Compact Memory`: **[高级]** 整理内存碎片（压缩）。
    - `Shell`: 执行任意自定义指令 (前缀 `shell:`)。

## 4. 💻 极客终端日志
传统的系统日志对极客来说太难检索。我们基于 WebSocket 实现了准秒级的实时日志终端，并对关键动作进行了高亮着色。

- **高亮标签**:
    - `[GHOST]`: 幽灵指令拦截记录。
    - `[AUTO]`: 自动化规则命中与动作执行。
    - `[SYSTEM]`: 系统级核心事件。
- **功能**: 支持关键字过滤、自动滚动以及连接状态监控。

## 5. 👻 幽灵短信指令 (Ghost Actions)
即使在 WebUI 无法连接的情况下，您仍可通过向设备发送特定格式的短信进行隐秘控制。

- **触发签名**: 必须以 `##UDX##` 开头。
- **指令列表**:
    - `##UDX##REBOOT`: 立即强制重启。
    - `##UDX##RESET_NET`: 强制重新注册网络/切换 APN。
    - `##UDX##CMD:ls -l`: 执行自定义命令（结果将记录在极客日志中）。
- **隐私**: 这些短信会被系统自动拦截，**不会**出现在普通的短信收件箱中。

## 6. 🛡️ 内存守护者 (Memory Guardian)
针对 128MB/256MB 的极低内存设备，我们开启了全方位的生存优化。

- **VM 调优**: 启动时自动调整 `vfs_cache_pressure`，促使内核更积极地回收目录缓存。
- **OOM 评分**: 核心服务 `oom_score_adj` 已被锁定为 `-500`，确保系统在崩溃边缘优先牺牲耗电进程，保住管理入口。
- **流控**: 所有的 WebSocket 缓冲区都被严格限制在 200 条以内，杜绝了长时间挂机带来的内存溢出。
